Posted on 2022-06-10 In Machine Learning Views: Symbols count in article: 2.9k Reading time ≈ 5 mins.

摘要

这是对 Hands-on Machine Learning with Scikit-Learn, Keras, and TensorFlow 这本书的一系列读书笔记，由于我之前没有接触过机器学习，所以读这本书对我来说刚刚好。这一篇介绍其中的第一章，包括机器学习是如何定义的，哪些问题适合用机器学习来解决以及机器学习的一些术语，大佬就请绕道吧~

PS: 本书链接在文章末尾，需要的同学自取✌

机器学习是什么？

书中谈到机器学习是计算机编程的科学(和艺术)，机器学习使得计算机可以通过数据来学习。这是一种看上去很清晰但是又感觉比较模糊的定义，关键是如何定义计算机“学习”呢？

设想这样的一个过程G，这个过程G需要完成一个目标T, 我们为了达到这个目标进行一系列的训练E, 一般先完成一系列的训练再去通过过程G达到目标T，我们设置一个评价指标P来测量最终达到目标T的效果。如果我们达到目标T的效果(这由P来刻画) 随着进行的训练E的增加而提高，那么就说这个过程是具备学习能力的，对于计算机来说，如果一个程序具备上面的特点，就可以说它具备“学习”的能力。

哪些问题适合使用机器学习来解决

  • (程序)需要手动调整或者有一长串规则的问题
  • 问题的背景是不断变化的环境
  • 问题的数据体量巨大

机器学习系统的类型(们)

书中对机器学习系统按照以下三种方式进行了划分

  • 按照是否需要人工参与划分为 有监督学习(supervised learning), 无监督学习(unsupervised learning) 以及半监督学习(semisupervised learning)。

    在有监督学习中，喂给机器学习算法的是带标签的数据；无监督学习则使用的是不带标签的数据；而半监督学习使用的则是一大部分的无标签数据和一小部分的带标签数据。

    有监督学习(或者说机器学习本身)实际上一般有两类应用，一类是分类(classification),另一类是回归(regression 这里回归可以简单理解为预测，虽然预测只是回归的其中一个方面)。另外，在机器学习中，attribute是数据类型，可以理解为字段，而feature一般来说是指字段和其取值。

    无监督学习的算法包括聚类，数据降维(数据降维可以提取特征，这种模式和重整化很像)，异常检测等。

  • 按照数据是一下子给足还是分批次给到机器学习算法而分为 批量学习(batch learning 数据一下子给足)和 在线学习(online learing 一点一点喂给机器学习算法)。

    Batch Learing 一般需要先在线下(offline)对数据进行学习，数据学习完毕之后才将其上线到生产环境中，如果需要对新的数据进行学习，那么就需要在线下对所有数据(新的&老的)进行学习。

    而Online Learing 一般也是线下学习数据，学习时并不是像Batch Learing那样一股脑把数据全部给到，而是将数据切片一部分一部分地喂算法(这也使得它可以适应新数据)，但是上线后遇到新数据时，可以一边运行一边学习新数据。

  • 按照是与数据进行比较还是建立回归模型分为 基于实例的学习(instance-based learning)和基于模型的学习(model-based learning)。

    Instance-based Learning 是说通过将新的数据和学习(训练数据)做对比，通过它们之间的相似程度来判断新的数据属于哪一类。自然这样的方法就需要设计合理的相似性度量。

    而model-based learing 本质上是要寻找feature之间的关联(回归就是干这个事儿的)，从而建立起数学模型，然后通过数学模型来预测新的值。这个数学模型中的参数是不确定的，需要我们通过一定的原则来求得，一般来说这里就要涉及到预测值和真实值之间的差异，需要定义一个cost function，通过最小化cost function 来求得这个模型中的参数。

机器学习面临的挑战

机器学习中我们可能遇到的问题主要来自于两个方面

  • 不好的训练数据

    不好的训练数据表现在以下几个方面：训练数据不足，缺乏有代表性的数据，数据中有过多的噪声，数据体现出与我们目标不相符合的特征。

  • 不好的模型

    好的模型在训练数据上表现优异，同时也具备良好的泛化能力。

    不好的模型一般有两种表现：

    • Overfitting(过拟合)

      overfitting 书中说得很清楚了，这里不再赘述。这里说一说避免overfitting的方法–正则化(regularization). 正则化是对模型的参数进行一定的限制(降低模型参数的自由度), 来使模型简单化以达到增强其泛化能力避免overfitting的方法。机器学习中用来刻画在训练中正则化程度的量(参数)叫做超参数(hyperprameter)，超参数是整个机器学习算法的参数，不同于模型的参数在训练的过程中渐进式地确定，超参数的值在开始训练之前就需要确定并保持下去。为了表示正则化和超参数的联系(但从名称上看是真的不容易将这二者联系起来)，我们一般的说法是 “正则化的超参数”

    • Underfitting(欠拟合)

      underfitting书中也说得很清楚了，这里从略。

测试和验证(Testing & Validating)

PS：这里主要是为了弄明白 测试，验证的概念以及训练和它们之间的关系，所以逻辑上和上面的小节会有点脱节。

为了使得训练之后的模型进行评估，我们需要对训练好的模型进行测试(Testing), 一般是将训练数据分为训练集(training set)和测试集(test set)，一般来说二者是八二开。将训练好的模型在测试集上进行测试，模型的计算值与测试集的标记值之间的错误率被称为泛化误差(generalization error).

有时我们需要对模型进行正则化以避免过拟合，由于正则化的程度是由超参数控制的，所以我们需要选择合适的超参数，那肯定就需要额外的数据来支持我们先确定超参数，因此就有了验证集(validation set). 验证集是从剩下的训练集(训练集减去测试集)中再分离出来的。具体来说是，在训练集上对模型进行训练(在多个模型下，选出最好的那个模型)，然后将模型在验证集上进行训练以找到最合适的超参数，接着在将得到超参数的模型在整个训练集上进行训练(训练集+验证集)得到最终的模型，最后通过测试集来评估整个模型(借助于泛化误差)。

如果验证集过大，那么就会导致真正用来训练的数据过少，这在我们需要对比多个模型时就十分不利，书上用了一个在我看来十分恰当的比喻：

It would be like selecting the fastest sprinter to participate in a marathon.

解决这个问题的一个比较优雅的方式是 **交叉验证(cross-validation)**，将验证集分为一小块一小块的, 然后将每个训练后的模型在每一个小块验证集上进行验证，通过平均值来描述一个模型在整个验证集上的表现，再通过比较平均值来找到最合适的那个模型。

更好地理解请移步：交叉验证与训练集、验证集、测试集
